Webhooks

        
Inbound webhooks allow external systems to trigger workflows or send messages to agents. All webhook requests require HMAC-SHA256 signature authentication using CYNTR_WEBHOOK_SECRET.

      


      

        

          Webhook Endpoints
        

        

          

            
Two webhook endpoints are available. Both require the X-Cyntr-Signature header with an HMAC-SHA256 signature of the request body, computed using CYNTR_WEBHOOK_SECRET.

            
            

              
1. Trigger Workflow

              

                
POST /api/v1/webhooks/trigger/{workflow_id}

                
Runs a registered workflow with an optional JSON payload.

              

              

                Headers:

                X-Cyntr-Signature: hex(HMAC-SHA256(body, secret))
              

              

                Body (JSON, optional):

                {"key":"value", ...}
              

              

                Response:

                {"status":"triggered","workflow_id":"...","run_id":"...", "payload":{...}}
              

            


            

              
2. Send Message to Agent

              

                
POST /api/v1/webhooks/agent/{tenant}/{agent}

                
Sends a message to an agent and returns the response immediately.

              

              

                Headers:

                X-Cyntr-Signature: hex(HMAC-SHA256(body, secret))
              

              

                Body (JSON):

                {"message":"..."}  or  {"text":"..."}  or  {"content":"..."}

                Falls back to raw body text if none of these fields match.
              

              

                Response:

                {"agent":"...","content":"...", "tools_used":["..."]}
              

            

          

        

      


      

        

          Authentication Setup
        

        

          

            
Important: Webhooks are disabled by default. To enable them, set the CYNTR_WEBHOOK_SECRET environment variable in your .env file before starting Cyntr.

            
            

              Example .env:

              CYNTR_WEBHOOK_SECRET=your-secure-random-secret-key
            


            
Signature Verification (example in Node.js):

            
const crypto = require('crypto');

function verifySignature(body, signature, secret) {
  const mac = crypto.createHmac('sha256', secret);
  mac.update(body);
  const expected = mac.digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}


            
Example curl request:

            
BODY='{"message":"hello"}'
SECRET='your-secret'
SIG=$(echo -n "$BODY" | openssl dgst -sha256 -hmac "$SECRET" -hex | cut -d' ' -f2)

curl -X POST http://localhost:8080/api/v1/webhooks/agent/default/my-agent \
  -H "Content-Type: application/json" \
  -H "X-Cyntr-Signature: $SIG" \
  -d "$BODY"
