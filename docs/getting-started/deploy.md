# Production checklist

This page is the minimum you should do before pointing real users or production cloud accounts at a Cyntr instance. Treat it as a sign-off list, not a tutorial.

## Security

- [ ] **TLS in front.** Cyntr listens on plain HTTP. Terminate TLS at a reverse proxy (Caddy, nginx, Cloud Load Balancer) before anything else.
- [ ] **API keys with scopes.** Rotate the autogenerated key from `cyntr init`. Issue scoped keys per consumer:
  ```bash
  CYNTR_API_KEYS=cyntr_read_xx:read,cyntr_agent_yy:agent,cyntr_admin_zz:admin
  ```
- [ ] **OIDC for humans.** Set `CYNTR_OIDC_ISSUER` and `CYNTR_OIDC_CLIENT_ID`. The dashboard will redirect to your IdP. See [concepts/multi-tenant.md](../concepts/multi-tenant.md).
- [ ] **Policy deny-by-default for production tenants.** Start with `deny` and `allow` only the (agent, tool) pairs you need. See [how-to/write-a-policy.md](../how-to/write-a-policy.md).
- [ ] **Sandboxed shell_exec.** If any agent has `shell_exec`, configure `shell_exec_policies.backend: docker` for that tenant. See [reference/config.md](../reference/config.md).
- [ ] **PII redaction on.** Default-on. Verify by chatting a fake SSN at an agent and confirming the audit log shows `<REDACTED>`.

## Reliability

- [ ] **Run two replicas.** Cyntr is stateless above the SQLite files; put both behind a load balancer and use shared storage (or external Postgres for sessions if you have it).
- [ ] **Health probes wired.** Kubernetes/Fly liveness on `/api/v1/system/health`, readiness on `/api/v1/system/ready`.
- [ ] **Graceful shutdown verified.** Cyntr handles SIGTERM cleanly (~5s drain). Confirm your orchestrator sends SIGTERM, not SIGKILL.
- [ ] **Disk monitored.** Audit + sessions + memory + usage + knowledge all share the working directory. Size for 30 days of audit retention; default TTL is 90d for sessions, 180d for memory, 365d for usage.

## Observability

- [ ] **Logs shipped.** Cyntr writes structured JSON to stdout; pipe to your aggregator.
- [ ] **OpenTelemetry wired.** Set `OTEL_EXPORTER_OTLP_ENDPOINT` to your collector. See [concepts/observability.md](../concepts/observability.md).
- [ ] **Prometheus scrape.** `/api/v1/metrics` returns OpenMetrics text. Scrape every 15s.
- [ ] **Alerts on policy denials.** A spike in `policy.decision="deny"` is usually a prompt injection or a misconfigured agent. Alert on it.

## Cost

- [ ] **Per-tenant quotas.** Set `quota.tokens_per_day` per tenant in `cyntr.yaml`. Without quotas, a runaway agent will spend your entire monthly budget in an afternoon.
- [ ] **Cheap default model.** Cost regression test: chat 100 turns and confirm spend matches the model's pricing card.

## Backups

- [ ] **Nightly backup.** `cyntr backup` snapshots all SQLite databases to a tarball. Off-host it.
- [ ] **Restore tested.** `cyntr restore <tarball>` on a clean host. Do this once before you trust the backup.

## Don't do this

- Do not put `shell_exec` on an agent that takes Slack input from outside your org. Prompt injection is real and you will eventually get a `rm -rf` attempt.
- Do not run with `policy.yaml` set to allow-all in production. The default sample is permissive on purpose; tighten it.
- Do not skip TLS because "it's behind a VPN." Internal threat actors are a thing.
