# Environment variables

Every env var the cyntr binary reads. Grouped by what it configures.

## Core

| Var | Default | Description |
|-----|---------|-------------|
| `CYNTR_API_URL` | `http://localhost:7700` | Client-side: where the CLI talks. |
| `CYNTR_API_KEY` | (autogenerated by `init`) | Primary API key. |
| `CYNTR_API_KEYS` | none | Multi-key form: `key1:scope1,key2:scope2`. Scopes: `read`, `agent`, `admin`. |
| `CYNTR_LOG_LEVEL` | `info` | `debug` / `info` / `warn` / `error`. |
| `CYNTR_DATA_DIR` | `.` | Where SQLite files are written. |

## LLM providers

| Var | Description |
|-----|-------------|
| `ANTHROPIC_API_KEY` | Claude. |
| `ANTHROPIC_MODELS` | Comma-separated list of Claude model IDs to register. |
| `OPENAI_API_KEY` | OpenAI. |
| `OPENAI_MODELS` | Comma-separated list. |
| `AZURE_OPENAI_API_KEY` | Azure AI Foundry. |
| `AZURE_OPENAI_ENDPOINT` | Azure endpoint URL. |
| `AZURE_OPENAI_DEPLOYMENT` | Deployment name. |
| `GEMINI_API_KEY` | Google Gemini. |
| `OPENROUTER_API_KEY` | OpenRouter (100+ models). |
| `OLLAMA_URL` | Local model server (e.g. `http://localhost:11434`). |

## Channels

| Var | Description |
|-----|-------------|
| `SLACK_BOT_TOKEN` | Slack bot token (xoxb-...). |
| `SLACK_SIGNING_SECRET` | For verifying inbound events. |
| `SLACK_ROUTES` | `C0123=agent-a,C0456=agent-b`. |
| `SLACK_USE_THREADS` | `true` to reply in threads. |
| `SLACK_APPROVAL_CHANNEL` | Channel ID for approval prompts. |
| `TEAMS_APP_ID`, `TEAMS_APP_PASSWORD` | Microsoft Teams. |
| `DISCORD_BOT_TOKEN` | Discord. |
| `TELEGRAM_BOT_TOKEN` | Telegram. |
| `WHATSAPP_ACCESS_TOKEN`, `WHATSAPP_PHONE_NUMBER_ID` | WhatsApp Business. |
| `EMAIL_SMTP_HOST`, `EMAIL_SMTP_PORT`, `EMAIL_USERNAME`, `EMAIL_PASSWORD` | Email. |
| `GOOGLE_CHAT_WEBHOOK_URL` | Google Chat. |
| `WEBHOOK_INBOUND_SECRET` | Shared secret for generic webhook adapter. |
| `SIGNAL_PHONE_NUMBER`, `SIGNAL_API_URL` | Signal. |

## Security

| Var | Description |
|-----|-------------|
| `CYNTR_OIDC_ISSUER` | OIDC issuer URL. |
| `CYNTR_OIDC_CLIENT_ID` | OIDC client ID. |
| `CYNTR_AUDIT_SECRET` | HMAC secret for audit hash chain. |
| `CYNTR_PII_REDACTION` | `true` (default) / `false`. |

## Cloud tools

| Var | Description |
|-----|-------------|
| `AWS_PROFILE`, `AWS_REGION` | Default AWS context for `aws` tool. |
| `AWS_CROSS_ACCOUNT_ROLES` | Comma-separated role ARNs for STS AssumeRole. |
| `GOOGLE_APPLICATION_CREDENTIALS` | GCP service account JSON path. |
| `KUBECONFIG` | Path to kubeconfig for `kubectl` tool. |

## Integrations

| Var | Description |
|-----|-------------|
| `GITHUB_PERSONAL_ACCESS_TOKEN` | For `github` tool + MCP server. |
| `JIRA_BASE_URL`, `JIRA_USER`, `JIRA_TOKEN` | For `jira` tool. |
| `BRAVE_API_KEY` | For `brave-search` MCP server. |
| `SERPER_API_KEY` | Optional Google search backend for `web_search`. |

## Observability

| Var | Description |
|-----|-------------|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | OTLP/HTTP collector base URL. When unset, instrumentation is no-op. |
| `OTEL_SERVICE_NAME` | Default `cyntr`. |
| `OTEL_TRACES_SAMPLER_ARG` | Sampling ratio (0.0–1.0). Default `1.0`. |

## Retention overrides

| Var | Default | Description |
|-----|---------|-------------|
| `CYNTR_SESSION_TTL` | `90d` | Session retention. |
| `CYNTR_MEMORY_TTL` | `180d` | Memory retention. |
| `CYNTR_USAGE_TTL` | `365d` | Usage record retention. |

## Packs

| Var | Description |
|-----|-------------|
| `CYNTR_PACK_LOOMFEED` | `1` to register the Loomfeed vertical pack (also via `packs.loomfeed: true` in cyntr.yaml). |
